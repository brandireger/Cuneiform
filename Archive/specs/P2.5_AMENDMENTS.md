## A2 — Supervision & discovery pool rules (applies corpus-wide)

- Bin docs are EXCLUDED from: duplicate-positive generation,
  Task A class labels, contrastive negative sampling (a bin fragment
  may secretly belong to any composition, including test-side ones —
  it is unlabeled, not negative), and all reported metrics' truth
  sets.
- Bin docs FORM the **discovery pool** (`discovery_pool.parquet`):
  doc_id, bin_cth, site, size stats, ATTESTED rendering pointer.
  Inference-time queries only. Deliverable P7 ranks proposed
  assignments of discovery-pool fragments to real compositions for
  expert verification. Never scored as ground truth.
- EXCEPTION: joined composite docs whose parent folder is a bin
  still yield valid JOIN pairs (the physical join is real regardless
  of catalogue assignment); keep those pairs, tagged
  `parent_is_bin=True`, reported both included and excluded.

## A3 — Join-pair tiering (`07_join_tiers.py`, amends join_pairs.jsonl)

Add per-pair fields:
- `tier`: A = no-overlap seam (n_shared_lines==0); B = minimal seam
  (1–2 shared lines); C = extended overlap (3+ shared lines).
- `geometry`: relabel — extended overlap = HORIZONTAL join candidate
  (break runs vertically; members split each line); no-overlap /
  single-line seam = VERTICAL join candidate (stacked). Replace the
  "duplicate_like" label everywhere; it misreads the physics.
- `join_type` already present (direct `+` vs indirect `(+)`); keep.
- For every tier-C pair, additionally emit an **exclusive-content
  variant**: both members with all shared lines DELETED. Store as
  parallel rendering (`members_exclusive=True`) alongside the full
  reconstruction. Evaluation policy (write into eval harness later):
  tier A is the headline physical-join metric; tier B secondary;
  tier C is reported ONLY in the exclusive-content variant as the
  honest number, with the contaminated full-reconstruction number
  shown alongside, labeled as upper bound. All strata × join_type
  reported — the over-explicit matrix stands.
- Degenerate-exclusive guard: after shared-line deletion, if either
  member retains < 8 attested signs OR < 2 lines, mark the pair
  `exclusive_untestable=True` — excluded from tier-C exclusive
  metrics, counted and reported separately (never silently included
  as trivial noise, never silently dropped).
- `join_tiers_report.md`: counts per tier × join_type × parent_is_bin,
  shared-line histograms, exclusive_untestable count, and 3 worked
  examples (one per tier) showing member renderings.

## A6 — Design commitment recorded for P5 (no implementation now)

**Seam scoring must NEVER assume contiguity.** Clay crumbles: even
direct `+` joins lose signs at the fracture face, and indirect `(+)`
joins are same-tablet pairs separated by an arbitrary lost span.
Therefore:
- Horizontal seam score = plausibility of [A-edge] [unknown-length
  masked span] [B-edge], via span-infilling (T5-sentinel-style
  variable-length mask), never next-token adjacency.
- Vertical seam score = same, with the mask spanning an unknown
  number of whole lines (anchor to gap t="line" / "bricht ab"
  events where present).
- Multi-row consistency = alignment at a CONSISTENT but UNKNOWN
  offset across row-pairs, not exact abutment.
- Evaluation stratum: direct `+` pairs test near-contiguous seams;
  indirect `(+)` pairs (213) are the designated held-out test of
  long-gap tolerance. Report both, never pooled-only.
Copy this section into CLAUDE.md's matrix-model section during the
P2.5 CLAUDE.md update.
